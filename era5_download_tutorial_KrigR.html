Downloading and Processing ERA5 Climate Data with KrigR in R

Author

Bimandra Djaafara

Published

July 5, 2025

Introduction

This tutorial provides a practical guide for downloading and processing ERA5 climate data using the KrigR package in R. While there’s already an extensive tutorial by the package creator here, along with GitHub documentation and a scientific publication, this tutorial aims to make the process more accessible and addresses some package updates not covered in the original documentation.

This tutorial is focused on specific use cases and doesn’t cover all package capabilities. I encourage reading the comprehensive documentation for a complete understanding of KrigR’s features.

Prerequisites

Before starting, you’ll need:

  1. An account on the Copernicus Climate Data Store
  2. Your API credentials (username and API key)
  3. Basic knowledge of R and spatial data
  4. Download all the data you needed for this tutorial from here

Downloading Climate Data

Temperature Data

Let’s start with downloading 2-meter air temperature data:

download_temperature_jkt <- tryCatch({
  CDownloadS(
    ## Variable and Data Product
    Variable = "2m_temperature",
    DataSet = "reanalysis-era5-land",
    ## Time Window
    DateStart = "2022-08-01 00:00:00",
    DateStop = "2022-09-30 23:00:00",
    TZone = "Etc/GMT+7",  # GMT+7 for Indonesia
    ## Temporal Aggregation
    TResolution = "day",
    TStep = 1,
    FUN = "mean",
    ## Spatial Limiting
    Extent = jakarta_extent,
    ## File Storing
    Dir = dir_data,
    FileName = "download_temperature_jkt",
    ## API User Credentials
    API_User = API_User,
    API_Key = API_Key
  )
}, error = function(e) {
  # Handle the common metadata writing error
  cat("Metadata writing failed, but attempting to load created file...\n")
  file_path <- file.path(dir_data, "download_temperature_jkt.nc")
  if (file.exists(file_path)) {
    cat("Successfully loaded data despite metadata error!\n")
    return(terra::rast(file_path))
  } else {
    stop("Data file was not created: ", e$message)
  }
})

Calculating Relative Humidity

Since relative humidity isn’t directly available, we calculate it using the Magnus formula:

# Function to calculate relative humidity
calculate_relative_humidity <- function(T_raster, Td_raster) {
  
  # Convert from Kelvin to Celsius
  T_celsius <- T_raster - 273.15
  Td_celsius <- Td_raster - 273.15
  
  # Calculate relative humidity using Magnus formula
  # RH = 100 * exp((17.625 * Td)/(243.04 + Td)) / exp((17.625 * T)/(243.04 + T))
  
  # Calculate saturation vapor pressure for dewpoint temperature
  es_td <- exp((17.625 * Td_celsius) / (243.04 + Td_celsius))
  
  # Calculate saturation vapor pressure for air temperature  
  es_t <- exp((17.625 * T_celsius) / (243.04 + T_celsius))
  
  # Calculate relative humidity (as percentage)
  RH_raster <- 100 * (es_td / es_t)
  
  # Set names to match input raster names
  names(RH_raster) <- paste0("RH_", names(T_raster))
  
  # Copy time information if it exists
  if (!is.null(terra::time(T_raster))) {
    terra::time(RH_raster) <- terra::time(T_raster)
  }
  
  return(RH_raster)
}

# Calculate relative humidity

Advanced Data Processing

Population-Weighted Aggregation

For more sophisticated analysis, you can aggregate climate data using population weights. This section shows how to create monthly summaries weighted by population density.

Helper Function for Population Alignment

# Function to align population resolution to climate data resolution
# Method from: https://climateestimate.net/content/weighting-schemes.html
align_population_to_climate <- function(pop_raster, climate_raster) {
  
  # Get resolutions
  pop_res <- terra::res(pop_raster)[1]  # 0.008333333
  climate_res <- terra::res(climate_raster)[1]  # 0.1
  
  cat("Population resolution:", pop_res, "\n")
  cat("Climate resolution:", climate_res, "\n")
  
  # Calculate resolution ratio
  res_ratio <- climate_res / pop_res
  cat("Resolution ratio:", res_ratio, "\n")
  
  # Step 1: Crop population to climate extent first
  climate_ext <- terra::ext(climate_raster)
  pop_cropped <- terra::crop(pop_raster, climate_ext)
  
  # Step 2: Check for grid alignment
  climate_xmin <- climate_ext[1]
  climate_ymin <- climate_ext[3]
  
  pop_ext <- terra::ext(pop_cropped)
  x_offset <- (climate_xmin - pop_ext[1]) / pop_res
  y_offset <- (climate_ymin - pop_ext[3]) / pop_res
  
  x_fractional <- x_offset - floor(x_offset)
  y_fractional <- y_offset - floor(y_offset)
  
  if (abs(x_fractional) > 0.01 || abs(y_fractional) > 0.01) {
    cat("Grid misalignment detected, disaggregating...\n")
    
    # Disaggregate by factor of 2 to align grids
    disagg_factor <- 2
    pop_disagg <- terra::disagg(pop_cropped, fact = disagg_factor) / (disagg_factor^2)
    pop_disagg <- terra::crop(pop_disagg, climate_ext)
    
    new_pop_res <- terra::res(pop_disagg)[1]
    agg_factor <- round(climate_res / new_pop_res)
  } else {
    cat("Grids already aligned, no disaggregation needed\n")
    pop_disagg <- pop_cropped
    agg_factor <- round(res_ratio)
  }
  
  # Step 3: Aggregate to match climate resolution
  if (agg_factor > 1) {
    pop_aligned <- terra::aggregate(pop_disagg, fact = agg_factor, fun = sum, na.rm = TRUE)
  } else {
    pop_aligned <- pop_disagg
  }
  
  # Step 4: Final alignment
  pop_final <- terra::crop(pop_aligned, climate_ext)
  
  if (!identical(terra::ext(pop_final), terra::ext(climate_raster))) {
    cat("Final resampling for perfect alignment...\n")
    pop_final <- terra::resample(pop_final, climate_raster, method = "near")
  }
  
  # Step 5: Set minimum population to avoid zero weights
  pop_final[pop_final == 0] <- 1
  pop_final[is.na(pop_final)] <- 1
  
  return(pop_final)
}

Monthly Temperature Summary Function

temperature_monthly_summary <- function(data, admin_shp, pop_raster = NULL){
  
  # Get dates and create month-year labels
  dates <- terra::time(data)
  months <- format(dates, "%Y-%m")
  unique_months <- unique(months)
  
  # Aggregate daily data to monthly means
  monthly_temp <- terra::tapp(data, months, fun = mean, na.rm = TRUE)
  
  # Set proper time stamps
  monthly_dates <- as.Date(paste0(unique_months, "-01"))
  terra::time(monthly_temp) <- monthly_dates
  
  # Calculate zonal statistics
  monthly_temp_list <- list()
  layer_name_to_date <- tibble(month_year = names(monthly_temp),
                               date = monthly_dates)
  
  for (i in 1:terra::nlyr(monthly_temp)) {
    layer_name <- names(monthly_temp)[i]
    
    if (!is.null(pop_raster)) {
      # Ensure correct CRS
      terra::crs(monthly_temp[[i]]) <- "EPSG:4326"
      pop_raster_aligned <- align_population_to_climate(pop_raster, monthly_temp[[i]])
      admin_shp <- sf::st_transform(admin_shp, "EPSG:4326")
      
      # Population-weighted and regular mean
      temp_values <- exactextractr::exact_extract(
        monthly_temp[[i]], 
        admin_shp, 
        fun = c('mean', 'weighted_mean'),
        weights = pop_raster_aligned,
        append_cols = TRUE
      )
      temp_values$temperature_celsius <- temp_values$mean - 273.15
      temp_values$temperature_celsius_pop_weighted <- temp_values$weighted_mean - 273.15
      
      # Extract total population for reference
      pop_values <- exactextractr::exact_extract(
        pop_raster,
        admin_shp,
        fun = 'sum',
        append_cols = TRUE
      )
      temp_values$total_population <- pop_values$sum
      
    } else {
      # Regular mean only
      temp_values <- exactextractr::exact_extract(
        monthly_temp[[i]], 
        admin_shp, 
        'mean',
        append_cols = TRUE
      )
      temp_values$temperature_celsius <- temp_values$mean - 273.15
    }
    
    temp_values$month_year <- layer_name
    monthly_temp_list[[i]] <- temp_values %>% left_join(layer_name_to_date)
  }
  
  # Combine all months
  data_summary <- do.call(rbind, monthly_temp_list)
  return(data_summary)
}

Monthly Precipitation Summary Function

precipitation_monthly_summary <- function(data, admin_shp, pop_raster = NULL){
  
  dates <- terra::time(data)
  months <- format(dates, "%Y-%m")
  unique_months <- unique(months)
  
  # Aggregate daily data to monthly sums (not means!)
  monthly_prec <- terra::tapp(data, months, fun = sum, na.rm = TRUE)
  
  monthly_dates <- as.Date(paste0(unique_months, "-01"))
  terra::time(monthly_prec) <- monthly_dates
  
  monthly_prec_list <- list()
  layer_name_to_date <- tibble(month_year = names(monthly_prec),
                               date = monthly_dates)
  
  for (i in 1:terra::nlyr(monthly_prec)) {
    layer_name <- names(monthly_prec)[i]
    
    if (!is.null(pop_raster)) {
      terra::crs(monthly_prec[[i]]) <- "EPSG:4326"
      pop_raster_aligned <- align_population_to_climate(pop_raster, monthly_prec[[i]])
      admin_shp <- sf::st_transform(admin_shp, "EPSG:4326")
      
      prec_values <- exactextractr::exact_extract(
        monthly_prec[[i]], 
        admin_shp, 
        fun = c('mean', 'weighted_mean'),
        weights = pop_raster_aligned,
        append_cols = TRUE
      )
      # Convert to mm

      pop_values <- exactextractr::exact_extract(
        pop_raster,
        admin_shp,
        fun = 'sum',
        append_cols = TRUE

      prec_values$total_population <- pop_values$sum
      
    } else {

        monthly_prec[[i]], 
        admin_shp, 
        'mean',
        append_cols = TRUE

}

Key Points and Best Practices

Variable Selection

  • Check available variables using Meta.Variables()
  • Note whether variables are cumulative (like precipitation) or instantaneous
  • Use appropriate aggregation functions (mean for temperature, sum for precipitation)

Time Zones

  • Always specify the correct time zone for your study region
  • Use “Etc/GMT+X” format (note the positive sign for eastern longitudes)

File Management

  • ERA5 downloads create both processed and temporary files
  • Consider deleting temporary files to save disk space
  • Use descriptive file names to avoid conflicts

Performance Considerations

  • Precipitation data takes significantly longer to process due to disaggregation
  • Consider using high-performance computing for large datasets
  • Download data in chunks for very large temporal or spatial extents

Data Quality

  • Always inspect your downloaded data before analysis
  • Check temporal coverage and spatial extent
  • Verify units and coordinate reference systems

Additional Resources

  • Population Data: LandScan Global Population Database
  • Weighting Methods: Climate Estimate Weighting Schemes
  • KrigR Documentation: Official KrigR Tutorial
  • ERA5 Variables: Check the Copernicus Climate Data Store for complete variable lists

Conclusion

This tutorial provides a foundation for downloading and processing ERA5 climate data using KrigR. The examples focus on Jakarta, Indonesia, but the methods are easily adaptable to other regions and time periods. Remember to:

  1. Start with small datasets to test your workflow
  2. Be mindful of download times and server capacity
  3. Consider population-weighted aggregation for human-relevant analyses
  4. Always validate your results before conducting analyses

The combination of ERA5’s comprehensive climate data and KrigR’s processing capabilities provides a powerful toolkit for climate research and applications.
